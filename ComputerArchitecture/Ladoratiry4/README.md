## Язык программирования

### Синтаксис (BNF)

```
program     ::= { definition | statement }
definition  ::= ":" name { statement } ";"
statement   ::= word | number | string_lit | "if" body ["else" body] "then"
              | "begin" body ("until" | "while" body "repeat")
              | "variable" name | "interrupt:" name
string_lit  ::= '."' text '"'
word        ::= any defined word or built-in
number      ::= [-] digit {digit}
```

### Встроенные слова

| Слово | Действие |
|-------|----------|
| `+` `-` `*` `/` `mod` | Арифметика |
| `and` `or` `xor` | Побитовые операции |
| `=` `<` `>` `<=` `>=` `0=` `not` | Сравнение, результат 0/1 |
| `dup` `drop` `swap` `over` `rot` `nip` | Стековые операции |
| `2dup` `2drop` | Двойные стековые операции |
| `@` `!` | Чтение/запись памяти |
| `emit` | Вывод символа |
| `key` | Чтение символа из порта |
| `>r` `r>` `r@` | Работа со стеком возвратов |
| `'` word `execute` | Execution token |
| `halt` | Остановка |
| `iret` | Возврат из прерывания |

### Семантика

- **Стратегия вычислений:** стековая, обратная польская запись (RPN). Операнды кладутся на стек, операции их потребляют.
- **Области видимости:** глобальные. Все `variable` и `:` определения видны везде.
- **Типизация:** нет. Всё — 32-битные знаковые целые.
- **Строковые литералы** `." text"` — Pascal-строки (length-prefixed), хранятся в секции данных.
- **Переменные** объявляются через `variable name`. `name` кладёт адрес на стек, `@` читает, `!` записывает.
- **Процедуры** объявляются через `: name ... ;`. Вызов по имени.
- **Execution token:** `' word` кладёт адрес процедуры на стек, `execute` выполняет по адресу.
- **Прерывания:** `interrupt: handler_name` объявляет обработчик. Обработчик должен заканчиваться `iret`.

## Организация памяти

### Регистры

```
r0–r28   — регистры общего назначения
r29 (sp) — stack pointer стека данных Forth
r30 (rp) — stack pointer стека возвратов
r31 (pc) — program counter (неявный)
flags    — Z (zero), N (negative)
v0–v3    — векторные регистры, 4×32 бит каждый
```

### Память команд (Гарвардская архитектура)

```
Instruction memory
+-----------------------------+
| 0x0000 : jmp <main>         |  точка входа
| 0x0001 : li r0, <handler>  |  адрес обработчика прерывания
|   ...                       |
| 0x0002 : __print_pstr       |  встроенные процедуры
| 0x0010 : __print_int        |
|   ...                       |
| 0xNNNN : main               |  пользовательский код
+-----------------------------+
```

### Память данных

```
Data memory
+-----------------------------+
| 0x0000–0x000F : зарезервировано
| 0x0001        : адрес обработчика прерывания (вектор)
| 0x0010–0x...  : статические данные (строки, переменные)
| 0x01FF        : вершина стека данных (растёт вниз)
| 0x02FF        : вершина стека возвратов (растёт вниз)
| 0xFFFE        : порт ввода (memory-mapped)
| 0xFFFF        : порт вывода (memory-mapped)
+-----------------------------+
```

### Как хранятся данные

- **Числовые литералы** до ±32767 — непосредственная адресация (`li rd, imm`). Большие числа — через `lui` + `addi`.
- **Строковые литералы** (`." text"`) — Pascal-строки в статической памяти: `[длина][char0][char1]...`.
- **Переменные** (`variable`) — ячейки в статической памяти. Адрес известен на этапе трансляции.
- **Процедуры** — в памяти команд. Вызов через `call`, возврат через `ret`.
- **Прерывания** — адрес обработчика хранится в `data_mem[1]`. При прерывании процессор сохраняет `pc` и прыгает на обработчик.

## Система команд

### Типы данных

32-битное знаковое целое. Один символ — одно машинное слово.

### Форматы инструкций

Все инструкции — 32 бита (RISC).

**R-тип** (регистр–регистр):
```
[31..26] opcode (6) | [25..21] rd (5) | [20..16] rs1 (5) | [15..11] rs2 (5) | [10..0] unused
```

**I-тип** (immediate/память/переходы):
```
[31..26] opcode (6) | [25..21] rd (5) | [20..16] rs1 (5) | [15..0] imm16
```

**V-тип** (векторные):
```
[31..26] opcode (6) | [25..24] vd (2) | [23..22] vs1 (2) | [21..20] vs2 (2) | [19..0] unused
```

### Набор инструкций

| Инструкция | Тип | Такты | Действие |
|-----------|-----|-------|----------|
| `add rd, rs1, rs2` | R | 2 | rd = rs1 + rs2 |
| `sub rd, rs1, rs2` | R | 2 | rd = rs1 - rs2, set flags |
| `mul rd, rs1, rs2` | R | 2 | rd = rs1 * rs2 |
| `div rd, rs1, rs2` | R | 2 | rd = rs1 / rs2 |
| `mod rd, rs1, rs2` | R | 2 | rd = rs1 % rs2 |
| `and/or/xor` | R | 2 | побитовые |
| `shl/shr` | R | 2 | сдвиги |
| `cmp rs1, rs2` | R | 2 | flags по rs1-rs2 |
| `mov rd, rs1` | R | 2 | rd = rs1 |
| `li rd, imm` | I | 2 | rd = sign_extend(imm16) |
| `lui rd, imm` | I | 2 | rd = imm16 << 16 |
| `addi rd, rs1, imm` | I | 2 | rd = rs1 + imm |
| `load rd, imm(rs1)` | I | 2 | rd = mem[rs1+imm] |
| `store rd, imm(rs1)` | I | 2 | mem[rs1+imm] = rd |
| `jmp imm` | I | 2 | pc = imm |
| `jz/jnz/jn imm` | I | 2 | условные переходы |
| `call imm` | I | 2 | rstack[--rp] = pc; pc = imm |
| `ret` | I | 2 | pc = rstack[rp++] |
| `halt` | I | 1 | остановка |
| `iret` | I | 2 | pc = saved_pc; in_interrupt = false |
| `vadd/vsub/vmul/vdiv/vcmp vd,vs1,vs2` | V | 2 | покомпонентные операции |
| `vload/vstore vd, imm(rs)` | V | 2 | загрузка/сохранение вектора |

### Ввод-вывод

Memory-mapped: `0xFFFE` — порт ввода, `0xFFFF` — порт вывода. Доступ через обычные `load`/`store`.

### Прерывания (trap)

Прерывания внутренние, приходят по расписанию тактов. При прерывании:
1. Симулятор записывает символ в `data_mem[0xFFFE]`
2. Сохраняется `pc`
3. Процессор переходит на адрес из `data_mem[1]`
4. Обработчик завершается `iret`
5. Вложенные прерывания маскируются

## Транслятор

### Запуск

```
python translator.py <source.forth> <output.bin> [output.dbg]
```

### Этапы трансляции

1. **Токенизация** — разбивка текста на токены, удаление комментариев `( ... )` и `\ ...`
2. **Компиляция** — однопроходная. Каждый токен немедленно генерирует инструкции.
3. **Разрешение меток** — fixup-список заполняется вперёдссылками, адреса подставляются в конце.
4. **Запись бинарного файла** — big-endian 32-бит слова. Сначала кол-во инструкций, потом инструкции, потом кол-во слов данных, потом данные.

### Формат бинарного файла

```
[4 байта] количество инструкций N
[N × 4 байта] инструкции (big-endian)
[4 байта] количество слов данных M
[M × 4 байта] данные (big-endian)
```

### Формат отладочного файла

```
<addr> - <HEXCODE> - <mnemonic>
0034 - 0C010010 - li r0, 16
```

## Модель процессора

### Запуск

```
python machine.py <program.bin> [input.txt] [log.txt]
```

### Формат файла ввода

Каждая строка: `<такт> <символ>`. Символ — либо в кавычках `'A'`, либо числом `65`, либо `\n`, `\0`.

```
100 'A'
200 'l'
300 \0
```

### DataPath

- Память команд и данных — раздельные (Гарвард), по 65536 слов.
- 32 регистра + 4 векторных (4×32 бит каждый).
- Флаги Z и N устанавливаются инструкцией `cmp` и `sub`.
- Запись в `0xFFFF` добавляет символ в выходной буфер.
- Чтение из `0xFFFE` возвращает текущий символ и сбрасывает порт в 0.

### ControlUnit

- Hardwired: каждая инструкция декодируется и выполняется напрямую в Python.
- Точность до такта: каждая операция ALU и каждый доступ к памяти — 1 такт (fetch + execute = 2 такта).
- Прерывания проверяются перед каждым fetch.

### Демонстрация векторных инструкций

`programs/vector_demo.forth` суммирует два массива из 4 элементов скалярным способом. Результат выводится на экран.

Для сравнения производительности запускается симулятор с логом:
```
python machine.py out/vector_demo.bin "" out/vector_demo.log
grep "HALT" out/vector_demo.log
```

### Демонстрация execution token

```forth
: square dup * ;
: apply ' square execute ;
: main 5 apply emit halt ;
```

`' square` кладёт адрес процедуры `square` на стек, `execute` вызывает её.
